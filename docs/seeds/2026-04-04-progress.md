## 기술적 문제 해결

### ERD Outbox 누락
- **문제**: PL 정합성 점검에서 ERD에 Outbox 테이블이 없음을 발견. Outbox 패턴을 설계에 명시했으나 ERD에 반영하지 않은 상태
- **해결**: BC별 전용 Outbox 테이블(ReservationOutbox, SupplierOutbox) 설계 후 ERD에 추가. 공통 1개 대신 BC별 분리한 이유는 소비 주기와 재처리 정책이 다르기 때문

### Money 타입 불일치
- **문제**: 설계 문서에서 금액을 int로 정의했으나, Supplier 통합 시 다통화(USD 소수점) 지원이 필요
- **해결**: 모든 금액 필드를 BigDecimal로 통일. DB 컬럼은 DECIMAL(19,4)로 설정하여 센트 이하까지 표현 가능

### 컨벤션 미현행화
- **문제**: pricing, location BC가 실제 설계에서 분리되었으나 컨벤션 문서에 반영되지 않은 상태
- **해결**: 컨벤션 문서에 pricing/location BC 분리를 현행화. 모듈 구조와 패키지 구조에 반영

### 도메인 코드 일관성 부족
- **문제**: VO 래핑 수준 불일치, compact constructor 패턴 불일치, Entity에 setter 존재
- **해결 방향**: Entity setter 전면 금지 + create() 팩토리 패턴으로 통일. marketplace 프로젝트의 persistence 구조를 참조하여 Entity 패턴 표준 수립
