## 2026-04-06 -- 예약 동시성 제어 의사결정

### 수행 내용
- 예약 생성 시 동시 재고 차감 전략 결정
- "가상 면접 사례로 배우는 대규모 시스템 설계" 호텔 예약 시스템 챕터(p.232~262) 분석
- 5가지 선택지(비관적 잠금, 낙관적 잠금, DB 제약 조건, Redis만, Redis+DB 2중 구조) 비교 후 "Redis + DB 2중 구조" 채택
- ADR-001 문서 작성 완료

### 의사결정
- **선택한 것**: Redis Lua Script + DB 원자적 UPDATE 2중 구조
- **이유**: 성능(Redis 밀리초 응답) + 정합성(DB 최종 보장) + 가용성(Redis 장애 시 DB 폴백) 세 가지를 모두 확보
- **대안 기각 사유**:
  - 비관적 잠금: 동시 처리량 낮고 다중 행 잠금 시 데드락 위험
  - 낙관적 잠금: 충돌 빈도 높으면 재시도 폭증으로 성능 저하
  - DB 제약 조건만: 정합성은 보장되나 성능 테스트 가산점 확보 어려움
  - Redis만: 가장 빠르지만 Redis 장애 시 서비스 불가 (SPOF)

### AI 활용
- 5가지 선택지 트레이드오프 분석에 AI 활용
- 책 원문 분석 및 프로젝트 상황에 맞는 적용 방안 도출에 AI 활용
- 최종 결정은 본인 판단 (성능 테스트 가산점 + Redis 장애 폴백 필요성 종합 고려)
