# AI 활용 기록

## 활용 개요
- **사용 도구**: Claude Code (CLI) + Playwright MCP + skill-creator
- **활용 방식**: 도메인 리서치 자동화, 크롤링 분석, 설계 검증, 문서 생성, 워크플로우 자동화
- **핵심 원칙**: AI가 생성한 내용을 그대로 사용하지 않고, 매 단계에서 본인의 판단을 더함

---

## 상세 기록

### 1. 요구사항 분석
- **활용 도구**: Claude Code (PDF 읽기)
- **AI의 역할**: 요구사항 문서를 읽고 필수/선택/기록 3개 카테고리로 구조화
- **본인의 판단**: 이 프로젝트는 결과물보다 "과정"이 중요하다고 판단. 단순 구현부터 시작하지 않고 의사결정 기록 체계를 먼저 설계하기로 결정
- **효과**: 전체 작업의 "선택과 집중" 전략 수립 기반 마련

### 2. OTA 도메인 리서치 (핵심 활용)
- **활용 도구**: Claude Code + Playwright MCP (browser_navigate, browser_snapshot)
- **AI의 역할**: 
  - 실제 OTA 3개 플랫폼에 접속하여 검색결과/상세페이지의 데이터 구조를 접근성 트리로 추출
  - 추출된 데이터를 필드별로 분류하고 플랫폼 간 비교 분석
  - 도메인 초안과 대조하여 누락/과잉/구조 차이 식별
- **본인의 판단**:
  - 크롤링 대상을 "해외 2개 + 국내 1개"로 선정: Supplier 통합을 이해하려면 해외 OTA가 공급자 역할을 하는 구조를 파악해야 했음
  - 검색 조건을 "서울, 1박, 2인"으로 통일: 플랫폼 간 비교 가능성 확보
  - AI가 추출한 필드 목록에서 구현 우선순위를 직접 판단 (Core 4개 / Design Only 3개 / Future 3개)
  - AI가 제안한 "배리어프리", "지속가능성 인증" 등은 현재 스코프에서 제외
- **효과**: 도메인 초안에서 놓친 핵심 개념 6개 발견 (성급, 결제방식, 대실, 가격 3단구조, 위치 평점 분리, Supplier 법적 정보)

### 3. OTA 도메인 전문가 가이드 작성
- **활용 도구**: Claude Code (문서 생성)
- **AI의 역할**: OTA 업계 표준 개념(Property, RoomType, RatePlan 등), 국내/해외 차이, 가격 구조, 용어 매핑을 참조 문서로 정리
- **본인의 판단**: AI가 제공한 업계 지식을 크롤링 데이터와 대조하며 검증. 예: AI는 "Rate Plan"을 설명했는데, 실제 크롤링에서 같은 객실에 4~5개 요금 행이 존재하는 것을 확인하여 RoomType:RatePlan = 1:N 관계의 실증적 근거 확보
- **효과**: 이후 크롤링 분석에서 "왜 이 필드가 중요한지" 업계 맥락을 붙일 수 있었음

### 4. 크롤링 자동화 스킬 개발
- **활용 도구**: Claude Code (skill-creator)
- **AI의 역할**: Playwright MCP 기반 크롤링 워크플로우를 반복 가능한 스킬로 체계화
- **본인의 판단**:
  - 특정 플랫폼명을 하드코딩하지 않고 범용적으로 설계
  - 국내/해외 구분을 스킬의 핵심 분석 축으로 설정
  - OTA 전문가 페르소나를 참조 문서로 분리 → 스킬이 업계 맥락을 가지고 분석하도록
- **효과**: 추가 플랫폼 분석이 필요할 때 동일한 프로세스로 재실행 가능

### 5. 과정 기록 자동화 체계 구축
- **활용 도구**: Claude Code (skill-creator)
- **AI의 역할**: 시드 데이터 → 최종 문서 변환 워크플로우 설계, 시드 포맷 정의
- **본인의 판단**: 
  - 기록을 "시드(원본)"와 "최종 문서(산출물)"로 분리. 시드는 작업 중 자유롭게 축적하고 최종 문서는 나중에 정제하는 2단계 구조 채택
  - AI 활용 기록에서 "AI 제안"과 "본인 판단"을 반드시 분리하도록 포맷 설계
- **효과**: 이 문서 자체가 이 체계로 만든 첫 번째 산출물
